# Инвертированный индекс на Python

## Описание проекта

Проект реализует построение инвертированного индекса по базе данных с генетической информацией в формате JSON-файлов. Инвертированный индекс позволяет быстро искать документы по ключевым словам. В проекте также проводится сравнение производительности данной реализации с аналогичным поиском через Elasticsearch и SQL.

## Используемые технологии

Python, Elasticsearch, SQLalchemy, JSON

## Алгоритм построения индекса

JSON файлы обрабатываются для определения применимых к ним правил диагностики генетических мутаций. По итогу обработки конкретный вариант гена приписывается к соответствующим правилам, что позволяет почти мгновенно составлять список нужных вариантов для выбранного правила, что является основным сценарием поиска по БД и в итоге кратно повышает пыстродействие (бенчмарки прилагаются).

## Сравнение производительности

### Результаты тестов

```
SQL + JSON parsing: 9946 results | 1.541889 sec
SQL LIKE + JSON parsing: 9946 results | 1.069736 sec
Inverted index: 9946 results | 0.000012 sec
Compressed index: 9946 results | 0.000481 sec
Elasticsearch: 9946 results | 0.022729 sec
```
Разовые временные затраты для составления индекса позволяет многократно повысить скорость поиска. Специфика обновления данных позволяет существенно улучшить отзывчивость системы, поскольку обновления БД и соответственно индекса можно проводить в ненагруженные часы, а основной сценарий поиска становится практически мгновенным.
