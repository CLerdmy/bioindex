# Инвертированный индекс для поиска ACMG-правил

## Описание проекта

Проект реализует построение индексной структуры на основе инвертированного индекса для поиска по базе генетических вариантов.

Источник данных - сформированная база `franklin.db` (см. комментарий `db/models.py`), содержащая информацию о генетических вариантах и ACMG-правилах классификации. Основная задача проекта - реализовать эффективный поиск по большому JSON-полю `rules`, содержащему результаты применения ACMG-правил к вариантам.

В рамках проекта были реализованы:

- собственный инвертированный индекс;
- два варианта сжатия индекса:
  - `Delta Encoding`;
  - `Elias Gamma Encoding`;
- поиск через `Elasticsearch`;
- сравнение производительности различных подходов к поиску:
  - SQL Full Scan;
  - SQL LIKE;
  - Inverted Index;
  - Compressed Index;
  - Elasticsearch.

## Используемые технологии

Python, C, Docker, Elasticsearch, SQLalchemy, JSON, pickle

## Алгоритм построения индекса

JSON файлы обрабатываются для определения применимых к ним правил диагностики генетических мутаций. По итогу обработки конкретный вариант гена приписывается к соответствующим правилам, что позволяет почти мгновенно составлять список нужных вариантов для выбранного правила, что является основным сценарием поиска по БД и в итоге кратно повышает пыстродействие (бенчмарки(`docs/`) прилагаются).

## Структура проекта

- `benchmarks/` - скрипты для запуска и сравнения производительности подходов поиска
- `scripts/` - скрипты построения и обслуживания индексов
- `data/` - хранение базы данных и индексов
- `docs/` - документация и результаты тестирования
- `db/` - SQLAlchemy-модели и подключение к базе данных
- `tests/` - тестирование кода

## Сравнение производительности

`Compressed - Delta`:

```
SQL + JSON parsing: 9946 results | 1.541889 sec
SQL LIKE + JSON parsing: 9946 results | 1.069736 sec
Inverted index: 9946 results | 0.000012 sec
Compressed index: 9946 results | 0.000481 sec
Elasticsearch: 9946 results | 0.022729 sec
```

Разовые временные затраты для составления индекса позволяет многократно повысить скорость поиска. Специфика обновления данных позволяет существенно улучшить отзывчивость системы, поскольку обновления БД и соответственно индекса можно проводить в ненагруженные часы, а основной сценарий поиска становится практически мгновенным.

## Запуск проекта

#### Запуск контейнера

```bash
docker compose run --rm app /bin/bash
```

#### Построение индекса

```bash
python scripts/build_inverted_index_acmg.py
```

#### Построение сжатого индекса

```bash
python scripts/build_compressed_index_acmg.py
```

#### Загрузка данных в Elasticsearch

```bash
python scripts/load_elasticsearch.py
```

#### Сборка CGAMMA

```bash
cd /app/cgamma_src && pip install -e . --no-build-isolation
```

#### Проверка статистики индексов

```bash
python scripts/check_index_files.py
```

#### Запуск бенчмарков

##### Сравнение одиночных запросов

```bash
python benchmarks/benchmark_full.py
```

##### Тестирование AND-запросов

```bash
python benchmarks/benchmark_and_queries.py
```
